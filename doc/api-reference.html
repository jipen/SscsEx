    SscsEx.TransferDb

    SscsEx.TransferDb: Manage the ETS/DETS database where the transfers are recorded and the failed ones, kept. When the SSCS server inits, DETS (ie on disk) table is copied to the ETS (ie in memory) table. If there's some failed transfers remaining, they are then retryed. All operation are then done on the ETS table and when (or if) the server terminate, the ETS table is saved back to the DETS table.
